Online Bookstore

To create an online book store that allows 2 types of users (Customers & admin) to interact with the store. The website was deployed in the cloud using AWS with a MySQL database that runs in the cloud. A Java MVC model was used in developing the application with java servlets as the controller, DAO and MySQL as the model, and HTML/CSS and JS for view. The website consists of multiple pages with the main page displaying the available books with the ability to browse by category, search by title and add the book to the shopping cart. Clicking on the book would open a new page displaying information and reviews on the selected book with the option of adding additional reviews. The shopping cart page which could be accessed from the main page display all books in the shopping cart, along with the total cost of all books in the cart and the ability to remove items from the cart. Users can then proceed with their purchase this would prompt them to login or register to continue. The payment page consists of a form requesting all information needed to continue with the purchase such as shipping information and credit card details (no 3rd party payment service was used as it was a school project). There is also a analytics page which only admins can access displaying information such as books sold each month, top 10 visited/sold books. REST API and AJAX calls were used to GET/POST information from/to the DB.

Online Attendance System

To create an online school attendance system that allows 4 types of users (admin, secretary, parents and teachers) to interact with a website in order to produce and manage the attendance records for all schools with the school board. A MERN stack was used in developing the application with React as the front end, express and node as the backend and MongoDB for the database. The website consists of a single page making use of multiple components to change the view. Each type of user can login allowing them to perform specific actions. An admin account can create and assign a secretary account to a school. A secretary account can create a classroom within their assigned school, create and assign a teacher/student to a classroom, and link parents to students’ profiles. A teacher can record the attendance of students within a classroom. A parent account can view the attendance record of the linked student profile. A software requirements specification document was created before development to document the overall descriptions, requirements, features of the application along with the use case, activity and class diagrams for the software.

Rock-Paper-Scissor Smash

Virtual Reality provides an immersive experience for users to interact with by simulating a real or virtual environment in a virtual 3D space. There is a wide array of immersion equipment out there that provide various level of presence. The two Virtual reality device that I will be focusing on is the leap motion and the oculus rift. The oculus rift is a head-mounted display, which provides a tracking system offering the user full 360-degree perspective and delivered a 960 x 1080 display to each eye with a refresh rate of 60Hz allowing the user to experience an illusion that he is actually in the game. The leap motion on the other hand provides hand tracking, which consist of two cameras and three infrared LEDs to read/interpret 3D data and infer the positions of occluded objects, allowing users to experience realistic hand interactions and gestures. By pairing the two hardware’s together I will be able to develop a Rock-Paper-Scissor Smash game, which implements gestural control/interactions. This game will consist of an interactive VR gestural control game. It will make use of the Oculus to provide visual perception of the environment and the leap motion to provide somatosensory perception. The objective of the game is to stay alive as long as possible and achieve the highest score possible by eliminating the most enemy before you lose. There will be 3 different types of enemies that spawn randomly throughout the environment, which will provide points when eliminated. Making use of the leap motion the user can make 3 available hand gestures (rock, paper, and scissors), each enemy can only be eliminated by a specific hand gesture (i.e. Rock beats paper). To eliminate the enemy, one will use the associated hand gesture and hit/interact with the enemy. These enemies slowly approach the center (towards you), once they reach the center the game is over and you lose, the game will only end once you lose.

Crowd Detection - Social Distancing

Computer vision algorithms used for people detection have applications in crowd management and surveillance. Crowd management is important for public safety, especially now amidst the COVID-19 pandemic. The pandemic has resulted in worldwide economic and social issues. Health professionals recommend practicing social distancing as one of the most effective methods to slow the spread of the virus. Computer vision algorithms can assist with social distancing efforts by providing a means to alert authorities when violations on the permitted headcount within a space occur. Crowd counting and localization can also be useful when designing public spaces such as airports and malls, and in making decisions on how to manage crowds in these public spaces. In this project, we combined overlapping views from multiple cameras in a multi-step approach to locate people in sparse crowds and determine whether they were adhering to social distancing requirements (2 metres apart is considered safe). We used a convolutional neural network, YOLOv3, to detect people in sequential frames from multiple cameras. The detection’s were then projected to a top view ground plane via perspective transform using intrinsic and extrinsic camera parameters. Next we aggregated the projected detections; we tried two methods for this, using Gaussian Mixture Models, and Kernel Density Estimation to obtain the best estimates of each person’s position on the ground plane. Finally, we obtained a people count and calculated the distance between the people.

EduMate

To create an online e-learning website that allows students to learn and interact with other like-minded individuals. The website was hosted on the cloud using Heroku. A Django MVC was used in developing the application with Django/Python acting as the backend, HTML/CSS and JS as the frontend and sqlite3 for the database. The website consists of multiple pages with a landing page as the first thing they see when new users access the website. From there they can create a new account or login to a existing one. Students can then learn from a series of available resources ranging from math to science, chat with other students that or already logged in, visit the forum page to ask questions and answer questions from fellow peers or go to the quiz section to test their knowledge on what they learned.
